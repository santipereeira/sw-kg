En cuanto a la materialización de las tripletas con Morph-KGC, se detectaron errores de codificación y caracteres no válidos al generar el archivo en formato N-Triples (.nt). Para solventar este problema y garantizar la integridad del grafo, se optó por una estrategia de robustez técnica: se generó la salida en formato N-Quads (.nq) y se desarrolló un script de procesamiento en Python utilizando la librería rdflib. Este script permitió filtrar líneas corruptas y realizar una conversión limpia a Turtle (.ttl). Gracias a este flujo, se logró un grafo final de 1.359 tripletas totalmente compatible con motores de inferencia y herramientas de edición como Protégé, integrando además enlaces externos a Wikidata como recursos de tipo IRI para potenciar la naturaleza Linked Data del proyecto."
