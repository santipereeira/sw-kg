Diseñamos a mano los mappings individualmente, y los pusimos en comun en clase. Una vez listos, utilizamos el comando **python3 -m yatter -i mapping.yarrrml.yaml -o mapping.rml.ttl** para obtener el RML, a continuación utilizamos el config.ini (disponible en esta carpeta) y el comando **python -m morph-kgc config.ini** para generar el RDF. A modo de observación, tuvimos varios problemas en la creación del RDF porque los nombres que elegimos para la ontología no se correspondian con los del csv, entonces la decisión que tomamos fue cambiarle los nombres al csv.
